機械学習の勉強

Permalink: 2015-01-05 17:05:00+09:00 by ishikawakouji in Machine Learning tags: machine learning 0xdata H2O

H2Oとは

0xdataが提供している機械学習のテストをWebのUIで比較的簡単にやってみることができるツール。

サーバはJavaで動作。Web以外にサーバへのアクセス方法がAPIとして提供されているらしいが、 まだ、その内容は公開されていない感じ。

また、オンメモリで動いているらしいので、起動時のJREのメモリの与え方で、限界が 決まるかもしれない。

ダウンロードで動かす

配布元はこちら。

http://0xdata.com/

適当なものをダウンロードしてくる。Javaの環境は別途構築しておく。

起動は、こんな感じ。

java -Xmx1g -jar h2o.jar

オプションは、一例。この場合の Xmx1g はヒープサイズを1GB確保している。

ブラウザからアクセスするには、ローカルをつつく。

http://localhost:54321/

外部からのアクセスを許可していれば、ネットワーク上のブラウザから 利用することがたぶん可能だが、やっていない。

軽く使い方

RandomForest や DeepLearning がさらっとできる。

ページの上部にメニューがあるので、主にここから動かす。

データの用意

ローカルにデータを準備しておく。これをサーバにアップロードする。 データは、ZIPで圧縮しているものも利用できるようだ。

Data/Upload でUpload画面になる。アップロードしたいファイルを 選択すればいい。

次に、 Parseする。内部形式のデータに変換しているものと思われる。 Uploadが完了すると、自動的に遷移する。

CSVとかだと、自動的に内容を判断してくれる。判断した結果に問題が なければ、そのまま Submit する。内部的には、Uploadしたファイル名の 拡張子を hex にした名前で登録されており、データを参照する key として利用できる。

モデルを動かす

メニュー内の Model を選択すると、機械学習として利用できる 各種モデルが並ぶ。ここから適当なものを選択する。

おもな共通入力部分を示す。

  • source には、パースした内部形式データ(hex)を指定する。 補完してくれるので、入力は比較的簡単。
  • response には、データ内の出力値部分を指定する。CSVで1行目にラベル が付いているようであれば、コレが選択肢に並ぶ。
  • 赤いアスタリスクが必須項目のようだ。
  • テスト用のデータがあれば、 validation に内部形式データ(hex)を 設定する。

まずは、デフォルトで動かしてみる。あとは、各種パラーメータを いろいろ修正してやってみるのは、他のツールと同じだろう。
