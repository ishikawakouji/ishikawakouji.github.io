機械学習の勉強

Permalink: 2014-11-28 15:50:00+09:00 by ishikawakouji in Machine Learning tags: machine learning Building Machine Learning System with Python 実践 機械学習システム

特徴量

学習データを意味のある数値に変換しなければいけない。

テキストの類似度の場合は、例えば単語の出現回数。これを各単語のベクトルとみなして、距離を求める。

テキストの類似度で使っている技術

  • 単語ごとの出現回数を数える：bag-of-words
  • 出現回数ベクトルを正規化する：出現回数によってベクトルの距離がかわるのがおかしい場合
  • 重要度の低い単語の削除：ストップワード
  • 語形変化を戻す：ステミング(stemming)、NLTK(Natural Language Toolkit)
  • 学習データ全体での出現度で単語を差別化する：TF-IDF(team frequency - inverse document frequency)、 対象の文章での出現数を他の文章での出現数で割ることで、出現数に差がある単語を特徴量として浮き立たせる

クラスタリング

特徴の似ている文書を分類する。同じような特徴ベクトルをフラットクラスタリングの一つ KMeansでやっている。

文章の類似度は最終的に特徴ベクトルの似ている順に並べる。そのため、クラスタリングは 新しい文章に対するベクトルの比較対象の数を減らすだけの目的に利用されており、 クラスタリングそのものはあまり重要ではないことに、注意する。

本の中でも、完璧なクラスタリングはできないと書いてある。
